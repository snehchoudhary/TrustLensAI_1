Design a modern, clean, and futuristic web + browser extension UI for a product called “TrustLens AI 2.0”.

Goal:
Create a real-time digital trust verification platform that analyzes online content and provides a Trust Score with explanations.

Target Users:
Students, researchers, journalists, and everyday internet users.

Style:
- Minimal, clean, modern UI
- Dark + light mode support
- Colors:
  - Green (trusted)
  - Yellow (warning)
  - Red (danger)
- Use glassmorphism or soft shadows
- Smooth animations and transitions
- Professional startup-level design

---

Main Screens to Design:

1. 🏠 Dashboard Page
- Search bar: “Paste URL or enter claim…”
- Upload button for image/video
- Trust Score card (big circular indicator)
- Score example: 72% (color-coded)
- Breakdown:
  - Source credibility
  - Content authenticity
  - Context accuracy
- Show “Why this score?” expandable section

---

2. 🔍 Analysis Result Page
- Display analyzed content (news/article/image)
- Trust Score prominently at top
- Sections:
  - Source Analysis (verified/unverified)
  - Claim Verification (true/false/misleading)
  - Context Check (matched/mismatched)
- Highlight suspicious parts in red
- Show explanation in simple language

---

3. 🧾 Claim Verification UI
- Input field: “Highlight or enter a claim”
- Output:
  - Verdict (True / False / Misleading)
  - Confidence %
  - Explanation
- Add visual indicators (icons, badges)

---

4. 🖼️ Image Verification Screen
- Upload image area
- Show:
  - Reverse image results
  - “Found in X sources”
  - Timeline of usage
- Flag reused or manipulated images

---

5. 🧩 Browser Extension UI (IMPORTANT)
- Small popup design
- Shows:
  - Trust Score badge
  - Quick summary
  - “View Details” button
- Floating indicator on web pages (green/yellow/red shield)

---

6. 📊 Trust Breakdown Visualization
- Graphs or bars showing:
  - Source reliability
  - Authenticity score
  - Context match
- Interactive hover explanations

---

7. ⚙️ User Preferences Page
- Set priorities:
  - Accuracy vs Speed
  - Strictness level
- Toggle notifications

---

UX Requirements:
- Extremely intuitive (no learning curve)
- Clear call-to-actions
- Use icons for clarity
- Show real-time feedback/loading states
- Include empty states and error states

---

Extra Features (if possible):
- “Why should I trust this?” button
- AI explanation panel (chat-like)
- Save history of checked content

---

Output:
- High-fidelity UI screens
- Consistent design system
- Components reusable (cards, buttons, badges)
